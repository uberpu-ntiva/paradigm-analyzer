Technical & Process Improvement Proposal

SCA Price Activation Modernization

Transitioning from manual, file-based activation to a state-driven, visual, and automated ecosystem.

Proposed Improvements

Scope: Enhancements to tracking, visibility, and automation logic. "Redefining the business" is out of scope.

1. CRM Activation Tracking Layer

wysiwyg

Technical Change

Implementation of a dedicated ActivationActivity entity within CRM. This layer will ingest logs directly from the SCA Bot (via API or log parsing), replacing disparate Excel files and email threads with structured, queryable records for every activation attempt.

Measurable Benefits

  • Eliminates Manual Triage: Removes the need to manually cross-reference email logs with CRM contracts.
  • Auditability: Provides a 100% trace of activation history (Submission ? Success/Failure ? Retry).
  • Data Integrity: Prevents "Contract Not Found" errors by validating against CRM GUIDs before submission.
Effort: 3 Months
Resources: 1 CRM Dev, 1 Analyst (50%)
Software Cost: Existing CRM License

2. Coverage Heatmap Visualization

grid_view

Technical Change

Development of a visual grid component (Heatmap) that renders the Activation Status across the multi-level Account Hierarchy (Corp ? Member ? Sub-Member). Uses data from the "Virtual Activation Engine" to color-code nodes (Green=Active, Red=Gap, Yellow=Conflict).

Measurable Benefits

  • Visibility: Instantly identifies activation gaps and inheritance breaks (e.g., T1 vs T2 conflicts) without Excel vlookups.
  • Speed to Action: Reduces time to identify "missing sites" from hours to seconds.
  • Compliance: Highlights "ghost" activations where a facility should be active but isn't.
Effort: 4 Months
Resources: 1 FE Dev, 1 UX (50%)
Software Cost: None (Internal Hosting)

3. Virtual Activation State Engine

memory

Technical Change

Creation of a logic middleware that computes the State Differential (Actual vs. Preferred) before pushing to CRM. It acts as a "Virtual Record" system, holding the computed state of what should happen vs. what is happening in SCA.

Measurable Benefits

  • Automated Correction: Logic can automatically trigger "Retry" or "New Site" flows without human intervention.
  • Reduced Errors: Filters out redundant requests (e.g., "Already Price Activated") before they clutter the logs.
  • Predictive: Allows simulation of activation runs (Preferred State) before execution.
Effort: 5 Months
Resources: 1 Backend Dev, 1 Arch (50%)
Software Cost: Cloud Compute (Azure/AWS)
